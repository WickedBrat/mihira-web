# Native Sign In with Apple Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the Safari-based Apple OAuth flow with native Sign In with Apple on iOS, still authenticating through the existing Supabase Auth setup, while keeping Android/web on the current web OAuth flow.

**Architecture:** A new isolated module (`features/auth/nativeAppleSignIn.ts`) owns nonce generation and the `expo-apple-authentication` → `supabase.auth.signInWithIdToken` exchange. `useSignIn.ts` branches `signInWithApple()` on `Platform.OS === 'ios' && isNativeAppleSignInAvailable()`, calling the new module when true and falling back to the existing `signInWithProvider('apple')` OAuth path otherwise. No UI components change.

**Tech Stack:** Expo SDK 54, `expo-apple-authentication`, `expo-crypto`, `@supabase/supabase-js` (`signInWithIdToken`), Jest + `@testing-library/react-native`.

## Global Constraints

- Spec: `docs/superpowers/specs/2026-07-11-native-apple-signin-design.md`
- `ios/` and `android/` native folders are committed directly to git and are **not** regenerated by `expo prebuild` automatically — the entitlements file must be hand-edited in addition to the `app.config.js` flag.
- No changes to `OAuthButton.tsx`, `ProfileAuthSheet.tsx`, `app/onboarding/step-17.tsx`, `app/onboarding-new/step-22.tsx`, `app/(tabs)/profile.tsx`, `app/oauth-native-callback.tsx`, or `lib/auth.tsx` — they all call `signInWithApple()` from `useSignIn`, which is the only branch point.
- Native Apple auth must exchange through the existing Supabase Auth session (`signInWithIdToken`) — no separate auth backend or session store.
- User-cancelling the native sheet (`error.code === 'ERR_REQUEST_CANCELED'`) must return `false` silently — no error toast. Any other failure must use the existing `showToast({ type: 'error', ... })` pattern already used for OAuth failures.
- New Expo module dependencies must be added via `npx expo install <package>` (resolves the correct version for the installed Expo SDK) — never hand-pick a version number.
- All new/modified code lives under `mobile/` (the Expo app root); paths below are relative to `mobile/` unless stated otherwise.

- [ ] **Step 3: Implement `nativeAppleSignIn.ts`**

Create `mobile/features/auth/nativeAppleSignIn.ts`:
```ts
import * as AppleAuthentication from 'expo-apple-authentication';
import * as Crypto from 'expo-crypto';
import { getSupabaseClient } from '@/lib/supabase';

async function generateNonce(): Promise<{ rawNonce: string; hashedNonce: string }> {
  const randomBytes = await Crypto.getRandomBytesAsync(32);
  const rawNonce = Array.from(randomBytes)
    .map((byte) => byte.toString(16).padStart(2, '0'))
    .join('');
  const hashedNonce = await Crypto.digestStringAsync(
    Crypto.CryptoDigestAlgorithm.SHA256,
    rawNonce
  );
  return { rawNonce, hashedNonce };
}

async function patchFullNameMetadata(
  fullName: AppleAuthentication.AppleAuthenticationFullName | null
) {
  if (!fullName) return;

  const name = [fullName.givenName, fullName.familyName].filter(Boolean).join(' ').trim();
  if (!name) return;

  try {
    const { error } = await getSupabaseClient().auth.updateUser({ data: { full_name: name } });
    if (error) console.error('[nativeAppleSignIn] full name patch error', error);
  } catch (error) {
    console.error('[nativeAppleSignIn] full name patch error', error);
  }
}

export async function isNativeAppleSignInAvailable(): Promise<boolean> {
  return AppleAuthentication.isAvailableAsync();
}

export async function signInWithAppleNative(): Promise<void> {
  const { rawNonce, hashedNonce } = await generateNonce();

  const credential = await AppleAuthentication.signInAsync({
    requestedScopes: [
      AppleAuthentication.AppleAuthenticationScope.FULL_NAME,
      AppleAuthentication.AppleAuthenticationScope.EMAIL,
    ],
    nonce: hashedNonce,
  });

  if (!credential.identityToken) {
    throw new Error('Apple did not return an identity token.');
  }

  const { error } = await getSupabaseClient().auth.signInWithIdToken({
    provider: 'apple',
    token: credential.identityToken,
    nonce: rawNonce,
  });
  if (error) throw error;

  await patchFullNameMetadata(credential.fullName ?? null);
}
```
